## Executive Summary

Corpus: 70 evidence records × 11 active concepts = 770 pairs
Observed non-neutral assignments: 38 (35 true, 3 false)
Neutral assignments: 732 (95.1% of pairs)

**Finding:** The current lexical shadow prefilter cannot safely skip any pairs for this corpus.

At the minimum non-trivial threshold (0.01), the prefilter loses 5 of 38 observed assignments (13.2% false negative rate, 86.8% recall). At the default threshold (0.05), it loses 17 of 38 (44.7% false negative rate, 55.3% recall). No threshold achieves both meaningful skip rate (>0 records) and acceptable recall (≥95%).

Root cause: the LLM detects concepts through semantic implication, domain jargon, and financial context that keyword matching cannot capture. Sparse title-only records are especially vulnerable — the LLM infers concepts from short art-market headlines ('White Glove Sale', '$78M Richter') that contain none of the concept's definition keywords.

**Recommendation: C. Redesign prefilter**

No threshold achieves both meaningful record-level savings (>0 records skipped) and acceptable recall (≥95%). The current lexical keyword prefilter is fundamentally mismatched to the semantic reasoning the LLM uses for this corpus. Skipping must NOT be enabled. See redesign options in the recommendation section.

### Root Cause Categories

At threshold 0.10, 26 of 38 observed assignments are false negatives.

- **Sparse/title-only records** (11 FN): Short titles (6–16 words) that imply concepts through domain context ('White Glove Sale', '$78M Richter') without using the concept's vocabulary.
- **Zero keyword match** (5 FN): Evidence text contains none of the 27–42 concept keywords, yet the LLM correctly infers the concept from semantic context.
- **Weak single-keyword match** (21 FN): Evidence matches only one generic keyword ('market', 'auction', 'cultural') that is insufficient to cross the relevance threshold.

### Why Lexical Matching Fails Here

The LLM scorer operates on semantic implication and domain reasoning:

- 'Marian Goodman's $35.1M Richter Leads Christie's $162.7M Trio of Sales' → **TrophyBuyerDemand=TRUE**. LLM infers trophy buying from auction scale; 0 keywords matched.
- 'SFMOMA Announces Landmark Jacob Hashimoto Installation' → **InstitutionalValidationPremium=TRUE**. LLM infers institutional validation from museum context; 0 keywords matched.
- 'How Did Phillips Pull Off a $115.2M White Glove Sale?' → **SpeculativeDemandCollapse=FALSE**. LLM infers no collapse from 'white glove' (100% sold); 0 keywords matched.

A keyword prefilter cannot replicate this reasoning without access to the LLM's semantic understanding of market terminology, institutional names, and financial context.

### If Redesigning

A redesigned prefilter should address the root causes:

1. **Domain-specific indicator dictionaries**: Instead of extracting keywords from generic definitions, maintain per-concept dictionaries of domain jargon ('white glove', 'blockbuster', 'biennale'). These are the signals the LLM actually uses.
2. **Embedding-based similarity**: Replace lexical matching with dense vector similarity between evidence embeddings and concept definition embeddings. Would capture 'white glove' → TrophyBuyerDemand even without keyword overlap. Requires a sentence encoder (adds dependency; see IMPLEMENTATION_PLAN Phase 13).
3. **Record-level prior from prior runs**: Accumulate per-record neutral rates from past scoring runs. If a record has been all-neutral in N consecutive runs, skip it on the next run. Safe because it only skips records with historical evidence of irrelevance.
4. **Calibrated confidence filter**: Use the LLM's own confidence scores from prior runs to identify concept/evidence pairs that are reliably neutral. Re-score only pairs that exceeded confidence threshold in any prior run.
